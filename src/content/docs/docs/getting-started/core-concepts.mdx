Promptless automates documentation creation and maintenance through three interconnected components that work together to understand your codebase, gather context, and publish updates to your documentation platforms.

## How the Components Work Together

<section id="core-concepts-workflow" class="pl-site-steps is-compact" data-sticky-heading="false">
  <div class="pl-site-step-list">
    <article class="pl-site-step" id="triggers-initiate-the-process">
      <span class="pl-site-step-num">1</span>
      <h3>Triggers Initiate the Process</h3>
      <p>
        <strong>Triggers</strong> are the starting point for all documentation updates. They monitor various
        sources for events that might require documentation changes:
      </p>
      <ul>
        <li>
          <strong>Code Changes</strong>: Pull requests in GitHub, Bitbucket, or GitLab automatically trigger
          analysis when new features or fixes are introduced
        </li>
        <li>
          <strong>Team Communication</strong>: Slack messages and Microsoft Teams mentions can trigger
          documentation updates
        </li>
        <li>
          <strong>Support Patterns (Beta)</strong>: Zendesk tickets help identify gaps in documentation based on
          recurring user questions
        </li>
      </ul>
      <p>
        Learn more in <a href="/docs/configuring-promptless/triggers">Configuring Promptless → Triggers</a>.
      </p>
    </article>

    <article class="pl-site-step" id="context-sources-enrich-understanding">
      <span class="pl-site-step-num">2</span>
      <h3>Context Sources Enrich Promptless&apos;s Understanding</h3>
      <p>
        <strong>Context Sources</strong> enhance the AI&apos;s understanding by providing real-time access to your
        organization&apos;s data:
      </p>
      <ul>
        <li>
          <strong>Project Management</strong>: Linear issues and Jira tickets provide business context and feature
          requirements
        </li>
        <li>
          <strong>Code Repository</strong>: GitHub integration gives deep understanding of your codebase structure
          and recent changes
        </li>
        <li>
          <strong>Historical Data</strong>: Past documentation patterns and team workflows inform better suggestions
        </li>
      </ul>
      <p>
        Learn more in <a href="/docs/configuring-promptless/context-sources">Configuring Promptless → Context Sources</a>.
      </p>
    </article>

    <article class="pl-site-step" id="documentation-platforms-receive-updates">
      <span class="pl-site-step-num">3</span>
      <h3>Documentation Platforms Receive Updates</h3>
      <p>
        <strong>Documentation Platforms</strong> are where the final documentation updates are published:
      </p>
      <ul>
        <li>
          <strong>Git-hosted Platforms</strong>: Docs are stored in Git and synced to hosting providers like Fern,
          Mintlify, Docusaurus, and more
        </li>
        <li>
          <strong>Multi-platform Publishing</strong>: Single trigger can update multiple documentation repositories
          simultaneously
        </li>
      </ul>
      <p>
        Learn more in <a href="/docs/configuring-promptless/doc-collections/github-repos-docs-as-code">Configuring Promptless → GitHub Repos (Docs as Code)</a>.
      </p>
    </article>
  </div>
</section>

<Note>
Context sources are queried in real-time and no customer data is stored by Promptless, ensuring your sensitive information remains secure.
</Note>

## Advanced Configuration

For enterprise deployments, you can customize how these components interact:

- **Directory-specific Triggers**: Monitor only specific parts of your codebase
- **Team-scoped Context**: Limit context sources to specific teams or projects  
- **Multi-environment Publishing**: Separate staging and production documentation workflows
- **Custom Integration Patterns**: Self-hosted deployments support custom trigger and publishing workflows
