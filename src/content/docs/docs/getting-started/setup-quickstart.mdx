In 15 minutes, you'll have Promptless watching your repositories and communication channels for documentation opportunities. Within a day, you'll have your first suggestions to review.

## What You'll Set Up

The setup wizard connects three types of systems:

**Where Promptless watches for changes (triggers)**
- GitHub PRs and commits that might affect docs
- Slack conversations that reveal documentation gaps
- Support tickets that surface common questions

**Where your documentation lives**
- Your docs repository (any git-based platform)
- Promptless will learn your structure, style, and conventions

**Where Promptless gathers additional context**
- Project management tools (Linear, Jira)
- Internal wikis (Confluence)
- Anything that helps Promptless understand *why* changes are happening

## Before You Start

**Create your account** at [accounts.gopromptless.ai](https://accounts.gopromptless.ai)

**Check your access levels.** Connecting integrations may require admin permissions:

- **Slack**: Workspace admin to install the Promptless app
- **GitHub**: Organization admin or sufficient repository access
- **Jira/Confluence**: Atlassian admin permissions
- **Linear**: Workspace admin permissions

<Tip>
Don't have all the permissions yourself? No problem. The setup wizard lets you invite teammates to help. They'll receive an email and can pick up exactly where you left off. One person can connect Slack, another can connect GitHub, and so on.
</Tip>

## What Unlocks at Each Step

As you connect systems, Promptless can do more:

**After connecting Slack:**
- Anyone can @mention Promptless to request documentation work
- Promptless can notify your team about new suggestions in a channel you choose
- You can enable passive monitoring of specific channels

**After connecting your docs:**
- Promptless learns your documentation structure and patterns
- It can match the voice and style of your existing content
- Suggestions are formatted to fit your conventions

**After connecting GitHub as a trigger:**
- Every PR is analyzed for documentation impact
- Promptless comments directly on PRs when docs updates are needed
- Engineers see documentation feedback in their normal workflow

**After connecting context sources:**
- Suggestions include business context from tickets and issues
- Promptless understands *why* changes are being made, not just *what* changed
- Links back to original sources help reviewers validate suggestions

## After Setup

Once connected, Promptless immediately starts:

1. **Indexing your documentation** - Learning your structure, voice, and patterns (takes 5-30 minutes depending on size)
2. **Monitoring connected triggers** - Watching for events that might need docs attention
3. **Building context** - Understanding your product through the sources you've connected

Your first suggestions typically appear within 24 hours of setup—often sooner if there's active development.
