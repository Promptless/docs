Promptless provides powerful workflow management features that help teams organize, track, and collaborate on documentation suggestions. These tools make it easier to manage documentation updates across team members and coordinate releases.

## Suggestion Assignment

You can assign specific suggestions to team members for review and approval. This helps distribute documentation work and ensures the right people are reviewing relevant changes.

<Steps>
  <Step title="Assign a Suggestion">
    Navigate to the suggestion you want to assign and select a team member from the assignee dropdown. The assigned team member will be notified and can see all suggestions assigned to them.
  </Step>

  <Step title="View Assigned Suggestions">
    Use the assignee filter in the suggestions list to view all suggestions assigned to specific team members, including yourself.
  </Step>
</Steps>

## Labeling System

Labels help categorize and organize suggestions based on their purpose, priority, or target release. This makes it easier to track different types of documentation updates.

### Common Label Use Cases

<CardGroup cols={2}>
  <Card title="Release Planning" icon="regular calendar">
    Label suggestions with upcoming release versions (e.g., "v2.1", "Q3 Release") to track documentation changes for specific releases.
  </Card>

  <Card title="Content Types" icon="regular tags">
    Organize suggestions by content type such as "API Documentation", "User Guides", "Troubleshooting", or "Getting Started".
  </Card>

  <Card title="Priority Levels" icon="regular flag">
    Use priority labels like "High Priority", "Low Priority", or "Urgent" to help team members focus on the most important updates first.
  </Card>

  <Card title="Review Status" icon="regular eye">
    Track review progress with labels like "Needs Review", "In Progress", "Ready to Publish", or "On Hold".
  </Card>
</CardGroup>

### Managing Labels

<Steps>
  <Step title="Add Labels">
    Select one or more labels for a suggestion to categorize it appropriately. You can add multiple labels to provide detailed organization.
  </Step>

  <Step title="Create Custom Labels">
    Create custom labels that match your team's workflow and documentation processes.
  </Step>
</Steps>

## Filtering and Organization

The suggestions list provides powerful filtering options to help you find and manage documentation updates efficiently.

### Available Filters

<AccordionGroup>
  <Accordion title="Filter by Assignee">
    View suggestions assigned to specific team members or see all unassigned suggestions that need attention.
  </Accordion>

  <Accordion title="Filter by Labels">
    Filter suggestions by one or more labels to focus on specific categories, releases, or priorities.
  </Accordion>

  <Accordion title="Combine Filters">
    Use multiple filters together to create precise views, such as "High Priority suggestions assigned to John for the v2.1 release".
  </Accordion>
</AccordionGroup>

### Smart Filtering

The filtering system is designed to help you work efficiently:

- **Quick Access**: Commonly used filter combinations are easy to access
- **Clear Results**: Filtered views clearly show which criteria are applied
- **Persistent Filters**: Your filter preferences are remembered across sessions

## Best Practices

### Effective Assignment Strategy

- **Assign by expertise**: Route suggestions to team members with relevant domain knowledge
- **Balance workload**: Distribute assignments evenly across team members
- **Clear ownership**: Ensure each suggestion has a clear owner for accountability

### Label Organization

- **Consistent naming**: Use consistent label naming conventions across your team
- **Meaningful categories**: Create labels that reflect your actual workflow needs
- **Regular cleanup**: Periodically review and clean up unused or outdated labels

### Filtering Workflows

- **Daily reviews**: Use assignee filters to check your daily documentation tasks
- **Release planning**: Filter by release labels to prepare documentation for upcoming releases
- **Priority management**: Use priority labels to focus on the most important updates first
