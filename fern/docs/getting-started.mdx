# Welcome to Promptless

Promptless automatically updates your docs, saving your team time and improving your customers' experience.

## Before You Start

- Create an account at [accounts.gopromptless.ai](https://accounts.gopromptless.ai)
- If you have more than 200 articles in your docs, or if you use uncommon triggers or documentation platforms, contact our team at [hello@gopromptless.ai](mailto:hello@gopromptless.ai)

## Quick Setup Overview

Get Promptless running in 2 simple steps:

<Steps>
  <Step title="Set up integrations">
    Set up integrations for triggers, context sources, and documentation platforms
  </Step>
  
  <Step title="Create a project">
    Configure how and when Promptless should update your docs
  </Step>
</Steps>

## Step 1: Set up integrations

Visit the [Integrations page](https://app.gopromptless.ai/integrations) in your dashboard to connect:

<CardGroup cols={3}>
  <Card title="Triggers" icon="bolt">
    **GitHub/Bitbucket**: Auto-trigger from pull requests
    **Slack**: Manual triggers via @mentions or message actions
  </Card>
  
  <Card title="Documentation Platform" icon="book">
    **GitHub/Bitbucket repos**: Most common setup
    **ReadMe API**: For ReadMe-hosted docs not synced to GitHub
  </Card>

  <Card title="Enhanced Context" icon="search">
  Connect **Linear**, **Jira**, or **Slack** to provide Promptless with additional context for better suggestions.
  </Card>
</CardGroup>

<Tip>
The more context sources you connect, the more accurate Promptless suggestions can become.
</Tip>

## Step 2: Create a project

Visit the [Projects page](https://app.gopromptless.ai/projects) to create your first project.

### Trigger Configuration

Choose your trigger type:

<AccordionGroup>
  <Accordion title="GitHub/Bitbucket Triggers">
    Automatically runs when PRs are opened in specified source repositories.
    
    **Auto-publish option**: Enable to automatically create documentation PRs instead of requiring manual review.
  </Accordion>
  
  <Accordion title="Slack Triggers">
    Runs when you @mention Promptless or use Slack message shortcuts.
  </Accordion>
</AccordionGroup>

<Info>
Need both trigger types? Create separate projects with identical settings for each trigger.
</Info>

### Project Configuration Options

<ParamField path="Context Sources" type="optional">
  Select which Linear teams, Jira projects, or other sources to search for context
</ParamField>

<ParamField path="Excluded Directories" type="optional">
  Specify directories to exclude from analysis (internal docs, deprecated content, etc.)
</ParamField>

<ParamField path="Auto-publish" type="boolean">
  Automatically create PRs with documentation changes (GitHub/Bitbucket only)
</ParamField>

## What's Next?

<Note>
After creating a project, Promptless will index your documentation. This typically takes a few minutes, during which Promptless processes your documentation structure and content.
</Note>

Once setup is complete, Promptless will:
- Listen for trigger events from your configured sources
- Analyze changes and gather relevant context
- Generate documentation suggestions for your review

<Success>
You're all set! Promptless is now ready to help automate your documentation updates.
</Success>
