# GitHub Enterprise Setup

**For GitHub Enterprise Server and GitHub Enterprise Cloud users**

If your organization uses GitHub Enterprise Server (self-hosted) or GitHub Enterprise Cloud with specific security requirements, create a custom GitHub App within your enterprise environment.

<Warning>
This setup is required for GitHub Enterprise Server instances and GitHub Enterprise Cloud organizations that don't allow third-party GitHub Apps. For standard GitHub.com organizations, use the [standard GitHub integration](/docs/integrations/github-integration).
</Warning>

## Prerequisites

- GitHub Enterprise admin access to create and configure GitHub Apps
- Network access to Promptless endpoints (see [IP whitelisting](#ip-whitelisting))
- Ability to generate and securely store private keys and client secrets

## Step 1: Register New GitHub App

1. Navigate to your GitHub Enterprise admin settings
2. Go to "GitHub Apps" and click "New GitHub App"
3. Fill in the basic app information:

<Frame>
  <img src="https://promptless-customer-doc-assets.s3.amazonaws.com/docs-images/org_2lvkgU9erOFxYhtEVVC0ymPrPdF/b5b5ff40-d884-4086-acb6-4bb2973bfd97-github-enterprise-app-registration.png" alt="GitHub Enterprise App Registration" />
</Frame>

### Required App Details

Fill in the following fields in your GitHub App configuration:

- **GitHub App name** (required): Enter a descriptive name like "Promptless Enterprise" or "Promptless (Your Company Name)"
- **Description**: Use: "Promptless keeps your technical guides up-to-date using AI."
- **Homepage URL** (required): `https://gopromptless.ai`

Configure the following webhook settings in your GitHub App:

## Step 5: Configure Webhook Events

Select events that should trigger Promptless documentation updates:

<Frame>
  <img src="https://promptless-customer-doc-assets.s3.amazonaws.com/docs-images/org_2lvkgU9erOFxYhtEVVC0ymPrPdF/41ffdf0d-27c1-4927-b1fc-b0183443a066-github-enterprise-webhook-events-updated.png" alt="GitHub Enterprise Webhook Events" />

### Required Webhook Events

Select these events to ensure Promptless receives necessary notifications:

- **Pull Request** - Pull requests opened, updated, or closed
- **Pull Request Review** - Pull request reviews submitted
- **Pull Request Review Thread** - Review threads created or updated
- **Pull Request Review Comment** - Individual review comments made
- **Push** - Code pushed to repositories
- **Release** - Releases created or updated
- **Repository** - Repository settings changed

## Step 6: IP Whitelisting

Add Promptless IP addresses to your network whitelist:

<Info>
If your GitHub Enterprise instance is behind a firewall or uses IP restrictions, whitelist these IP addresses for Promptless to function.
</Info>

### Promptless IP Addresses

Add the following IP addresses to your whitelist:

```
3.143.177.103
3.131.121.250
3.13.184.175
18.223.104.40
```

<Warning>
These IP addresses are required for webhook delivery and API communication. Without them, Promptless cannot receive GitHub events or access your repositories.
</Warning>

<Info>
The primary egress IP address is `3.143.177.103` (NAT gateway). This IP appears in your GitHub Enterprise logs for API requests.
</Info>

## Step 7: Generate Credentials

After configuring all settings, generate the necessary credentials:

<Steps>
  <Step title="Generate Private Key">
    1. Scroll to the bottom of the GitHub App settings page
    2. Click "Generate a private key"
    3. Download and securely store the `.pem` file
    4. You'll need this key for Promptless configuration
  </Step>

  <Step title="Generate Client Secret">
    1. In "Client secrets", click "Generate a new client secret"
    2. Copy and securely store the generated secret
    3. You'll need this secret for OAuth authentication
  </Step>

  <Step title="Note the App ID">
    1. Find the "App ID" at the top of the settings page
    2. Record this ID for configuration
  </Step>
</Steps>

<Warning>
Store the private key and client secret securely. These credentials provide access to your GitHub Enterprise instance.
</Warning>

## Step 8: Install the App

1. After creating the GitHub App, install it to your organization or specific repositories
2. Choose which repositories Promptless should have access to
3. Complete the installation process

## Step 9: Connect Your GitHub Enterprise Integration

After configuring your GitHub Enterprise integration, connect it in your Promptless dashboard.

<Frame>
  <img src="https://promptless-customer-doc-assets.s3.amazonaws.com/docs-images/org_2lvkgU9erOFxYhtEVVC0ymPrPdF/8a5d1abd-3c56-4c5f-a7b1-340ecd794940-github-enterprise-connect-button.png" alt="GitHub Enterprise Connection Interface" />
</Frame>

### Completing the Connection

To establish the connection:

1. Log into your Promptless dashboard at [app.gopromptless.ai](https://app.gopromptless.ai)
2. Navigate to the Integrations page  
3. Locate the GitHub Enterprise integration tile
4. Click **Connect GitHub Enterprise** to establish the connection

<Info>
You'll need this information during connection:
- **GitHub Enterprise URL**: Your GitHub Enterprise instance URL
- **App ID**: The numeric ID of your GitHub App
- **Client ID**: Found in your GitHub App settings
- **Client Secret**: The secret from Step 7
- **Private Key**: The `.pem` file from Step 7
- **Installation ID**: Provided after app installation
</Info>

Once connected, you'll see confirmation that your GitHub Enterprise integration is active.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Webhook Delivery Issues">
    - Verify IP addresses are whitelisted
    - Check webhook URL is accessible from your GitHub Enterprise instance
    - Ensure SSL verification settings match your requirements
    - Review GitHub Enterprise logs for delivery errors
  </Accordion>

  <Accordion title="Authentication Problems">
    - Verify all URLs are correctly configured
    - Ensure private key and client secret are valid
    - Check App ID matches your GitHub App
    - Confirm installation completed successfully
  </Accordion>

  <Accordion title="Permission Errors">
    - Review that all required permissions are granted
    - Ensure app is installed to correct repositories
    - Verify webhook events are configured
    - Check users have appropriate repository access
  </Accordion>
</AccordionGroup>
