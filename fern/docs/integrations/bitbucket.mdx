# Bitbucket Integration

Promptless integrates with Bitbucket to automatically update your documentation when code changes are made. Connect your Bitbucket workspace and repositories. Monitor pull requests for changes that require documentation updates. Automate the documentation process.

## Installation

To set up the Bitbucket integration:

1. Navigate to the **Integrations** page in your Promptless dashboard
2. Click on **Connect** next to the Bitbucket option

<Tip title="Creating a Bitbucket App Password">
To create an app password with the right permissions:
1. Go to your Bitbucket account settings
2. Navigate to **App passwords** under **Access management**
3. Click **Create app password**
4. Give it a descriptive name like "Promptless Integration"
5. Select the following permissions:
   - **Repositories**: Read
   - **Pull requests**: Read and Write
   - **Webhooks**: Read and Write

### Directory-Specific Triggers

Configure Promptless to trigger only on changes to specific directories. This is useful for:

- Focusing on code that impacts user-facing features
- Monitoring API changes requiring documentation updates
- Ignoring internal tooling or test changes

To set up directory-specific triggers:

1. In your project configuration, select the Bitbucket repository
2. Under **Advanced Options**, enable **Directory-Specific Triggers**
3. Enter directories to monitor, separated by commas (e.g., `src/api, docs/reference`)

## Authentication Model

The Bitbucket integration uses username and app password authentication:

- **Username**: Your Bitbucket username for API authentication
- **App Password**: A secure token with specific permissions
- **Workspace Access**: Limited to the workspace you connect

Promptless only accesses repositories and actions you explicitly authorize.

## Repository Refresh

If you add new repositories after setup, refresh the repository list:

1. Go to the **Integrations** page
2. Find the Bitbucket integration
3. Click **Refresh Repositories**

## Webhook Management

Promptless automatically sets up webhooks in your Bitbucket repositories. These webhooks:

- Trigger Promptless when pull requests are created or updated
- Send only necessary information about changes
- Can be customized to focus on specific events

## Usage

Once configured, the Bitbucket integration works similarly to GitHub:

1. Pull requests in monitored repositories automatically trigger Promptless
2. Promptless analyzes changes to determine if documentation updates are needed
3. If needed, Promptless generates documentation changes
4. Promptless adds a comment to your pull request with a review link
5. Review and approve suggested updates in the Promptless dashboard

For more information, see the [Triggers](/docs/core-concepts/triggers) documentation.
